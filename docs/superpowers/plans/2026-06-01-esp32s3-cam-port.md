# ESP32-S3-WROOM CAM Port Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Retarget CamCar firmware from the classic ESP32-CAM to the Freenove ESP32-S3-WROOM CAM, with all hardware pins centralized in one `board_config.h`.

**Architecture:** A new `src/board_config.h` is the single source of truth for every GPIO (camera + drive peripherals). `Camera.h` and `CamCar.ino` consume it. Build switches to the `esp32s3` FQBN with OPI PSRAM. Classic ESP32-CAM support is dropped.

**Tech Stack:** Arduino / arduino-cli, esp32 core 3.3.x (pin-based LEDC), ESP32Servo, ESPAsyncWebServer, esp_camera (OV2640).

**Verification note:** No unit tests exist. Each task is verified by `make build` (full compile for the S3 FQBN). On-device functional verification (Task 7) requires the physical board and wired chassis.

### Task 1: Create the board-config header

**Files:**
- Create: `src/board_config.h`

- [ ] **Step 1: Write `src/board_config.h`**

```cpp
#ifndef BOARD_CONFIG_H
#define BOARD_CONFIG_H

// Hardware pin map for the Freenove ESP32-S3-WROOM CAM board.
// Single source of truth for all GPIO assignments (camera + peripherals).
// Camera pins verified against Freenove ESPHome config and Freenove GPIO notes.

// ---- Camera (OV2640, onboard -- no user wiring) ----
#define CAM_PIN_PWDN  -1
#define CAM_PIN_RESET -1
#define CAM_PIN_XCLK  15
#define CAM_PIN_SIOD   4   // I2C SDA
#define CAM_PIN_SIOC   5   // I2C SCL
#define CAM_PIN_D7    16   // Y9
#define CAM_PIN_D6    17   // Y8
#define CAM_PIN_D5    18   // Y7
#define CAM_PIN_D4    12   // Y6
#define CAM_PIN_D3    10   // Y5
#define CAM_PIN_D2     8   // Y4
#define CAM_PIN_D1     9   // Y3
#define CAM_PIN_D0    11   // Y2
#define CAM_PIN_VSYNC  6
#define CAM_PIN_HREF   7
#define CAM_PIN_PCLK  13

// ---- Drive peripherals (wired by user to these free GPIOs) ----
#define MOTOR_SPEED_PIN   1   // shared PWM enable for BOTH motor drivers
#define RIGHT_MOTOR_IN1  41
#define RIGHT_MOTOR_IN2  42
#define LEFT_MOTOR_IN1   40
#define LEFT_MOTOR_IN2   39
#define PAN_PIN          47   // pan servo signal
#define TILT_PIN         21   // tilt servo signal
#define LIGHT_PIN        14   // headlight LED (PWM)
#define STATUS_LED        2   // onboard LED (WiFi-connect blink)

#endif // BOARD_CONFIG_H
```

### Task 7: On-device smoke test (requires hardware)

**Files:** none (manual/hardware verification)

> Functional verification needs the physical board (connected at `/dev/ttyACM0`) and, for the drive test, the wired chassis. The camera/WiFi portion can be checked with just the board.

- [ ] **Step 1: Flash**

Run: `make upload`
Expected: esptool writes the image and `Hard resetting via RTS pin...`.

- [ ] **Step 2: Capture boot log** (non-interactive)

Run: `stty -F /dev/ttyACM0 115200 -echo; timeout 12 cat /dev/ttyACM0`
Expected: `BEGIN`, free-heap/flash lines, a WiFi connect/IP line (or "starting setup AP"), `HTTP server started`, and **no** "Camera initialization failed!".

- [ ] **Step 3: Browser test** — open the device IP, confirm video streams.

- [ ] **Step 4: Drive test (after chassis wired per the wiring table)** — confirm motors respond to direction + speed, pan/tilt servos move, light dims. Watch the serial log for LEDC/servo init warnings (the flagged contention risk).

- [ ] **Step 5: Record results** in the PR/branch description; note anything that needed adjustment (e.g. servo LEDC channel tweaks).
