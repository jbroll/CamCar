Pan / Tilt servos — wiring

GPIOServo signal
47Pan — signal lead
21Tilt — signal lead
 
 standard 3-wire hobby servo:
   signal (orange/white) ─► ESP GPIO  (47 pan · 21 tilt)
   +5V    (red)          ─► 5V rail   ← BEC, NOT the L298N reg
   GND    (brown/black)  ─► common ground

Power: servos pull big stall-current spikes — feed +5 V from a separate BEC (≥1–2 A) and share ground with the ESP & battery. Don't run them off the L298N's weak onboard 5 V regulator — it sags and resets the board. Add a 470–1000 µF cap across the servo +5 V rail to absorb the spikes.

Direction/range is set in firmware (cameraControl()); if a servo moves the wrong way, reverse it in code rather than re-pinning.

Reserved — never wire (N16R8)

33–37 octal PSRAM  ·  0 / 3 / 45 / 46 strapping
19 / 20 native USB  ·  43 / 44 UART0
48 onboard NeoPixel
Battery sense must use an ADC1 pin (GPIO 1–10); 1 & 2 are the only non-camera ADC1 pins.

Why the grouping matters: bare, unterminated header pins spread across the board detune/interfere with 2.4 GHz WiFi. Populate only the pins in use, keep them clustered, and leave the rest of the header empty.
